A. Fishbone Diagram (Problem Analysis of the Manual System)

Ang sumusunod na Cause-and-Effect (Fishbone) Diagram ay nagpapakita ng mga pangunahing problema ng kasalukuyang manwal na sistema ng barangay na naging basehan para sa pagbuo ng ProfiligSystem.

FISHBONE DIAGRAM: MANWAL NA SISTEMA NG BARANGAY
MANPOWER (Tao)
  • Kakulangan sa IT training ng mga barangay staff.
  • Mabagal na paghahanap ng records dahil manu-mano.
  • Nauubos ang oras sa paglipat-lipat ng pahina ng logbook.
MACHINES (Kagamitan)
  • Walang centralized database server.
  • Unstable o madalas walang internet connection.
  • Mabagal na mga lumang computer.
METHODS (Paraan)
  • Mano-manong paglista sa mga record logbooks.
  • Walang automated na pag-detect ng mga repeat offenders.
  • Mahirap i-verify ang pagkakakilanlan ng residente.
MATERIALS (Materyales)
  • Mataas na gastos sa papel at ink (printing ng physical records).
  • Madaling masira, mabasa, o mawala ang mga physical logbooks.
  • Kakulangan sa physical storage/cabinets para sa lumang records.
EFFECT (Epekto):
Mabagal, hindi sigurado (inaccurate), at madaling masirang sistema ng Barangay Profiling at Certification.


B. System Development Phases (SDLC Phase Table)

Ang sumusunod na talahanayan ay nagpapakita ng mga yugto (phases) sa pagbuo ng ProfiligSystem, na sumusunod sa Iterative o Agile Methodology.

PHASE ACTIVITY (Gawain) DESCRIPTION (Paliwanag)
Phase 1 Planning & Requirements Gathering Pagsasagawa ng interview sa mga barangay officials (Secretary at Admin). Dito inalam ang mga problema tulad ng kawalan ng internet, mabagal na paghahanap ng record, at pangangailangan para sa Offline-Ready na sistema.
Phase 2 System Design Pagbuo ng arkitektura ng system kabilang ang Database ERD, DFD, at Hierarchical Diagram. Dito idinisenyo ang UI/UX gamit ang Figma at ginawa ang logic para sa LocalStorage caching at Firebase sync.
Phase 3 System Development & AI Integration Pagsulat ng code (coding) gamit ang React, Node.js, at pag-integrate ng NVIDIA Llama 3.3 para sa AI pattern detection. Binuo rin dito ang Offline-First capability upang gumana ang system kahit walang internet.
Phase 4 Testing & Evaluation Pagsasagawa ng Unit Testing at Integration Testing. Pinagamit sa mga piling user ang system upang makakuha ng feedback gamit ang System Usability Scale (SUS) questionnaire upang masiguro ang kalidad bago ang final deployment.
Phase 5 Deployment & Maintenance Opisyal na pag-deploy ng Frontend sa Vercel at Backend sa Render. Sinigurado rin ang security (JWT at Hashing) upang maprotektahan ang data privacy ng mga residente ng Mamburao.
